Brief


As a Design and Marketing Associate for Simon Fraser University Graduate and Postdoctoral Studies one of my responsibilities was to design profile pages using Adobe Experience Manager (AEM). I was also responsible for updating the various web pages found in the GPS site such as the job page, event page, awards page, and had coded the template for the GPS news bulletin. Last minute profile submissions have piled up since winter break of 2020. Perhaps Graduate students had more time to submit it over the break. With over 60 profiles to be completed within a short time frame, the volume is overwhelming. My solution for this problem was to create a program to aid with the formatting. This generates a premade template for the profile pages. As a result of my innovation I was able to complete the profile pages at a faster rate with more consistency.



< process >

Process


My supervisor was proud of my innovations in accomplishing this task and required the Profile Helper to continue with operations within the department especially when there were large volumes of profiles requested. Unfortunately, there was a problem with my prototype Profile Helper. This version was built with the scripting programming language: Hypertext Preprocessor, which deals with web server functionalities and was not accessible to the department. I finally decided to use jQuery as an alternative programming language to build the Profile Helper. jQuery is a JavaScript library that can be used to create dynamic webpages. While using jQuery, I could generate profile templates just like before and accessible for my supervisor. This function was perfect for my Profile Helper as it allowed me to change string values by user input without the need of a server environment. $(selector).val() The method above allowed me to return what the user had inputted. $(selector).val(value) This method above allows the user to to set the value and can be set through the html tag

< brief >

Brief


Computer-related jobs are a major industry in Canada, America, Asia, and around the world. As a School of Interactive Arts & Technology major intending to pursue these types of fields we proposed a minimalistic yet concise job search website that focuses on computer science-related jobs in Canada.

Users can search by job title, company, city, province, or by postdate. Company representatives such as hiring managers can also visit our website to post new jobs. Through the comment sections, users will be able to make any inquiries to the hiring manager in regards to the posted job position. Registered users gain an additional feature as they can bookmark any job position to look into later at the home page.

This was a group project and I was responsible for creating the user registration form, dataset and user authentication along with the jobs populating in the search results.



Group members: Sorren Alex Jao & Sebastian Peng

< process >

Process


The data types of our dataset include the posting ID (int), job title (text), company (text), city (text), province (text), salary range (text), postdate (int), summary (text) and the job url (text). We used all the mentioned data types for our website. Users can search by job title, company name, and can filter the city, province, and postdate by radio button. Once the user receives their results the user can then select an item to view the summary of the job.


Entity Relation Diagram


The Member entity has four attributes: User_ID, email, password, and username. The User_ID is the primary key of each user. The username attribute is also unique to each member as they cannot have more than one account linked to their User_ID. Each user is able to set a password and username as they please. Users can have passwords and usernames that are identical to others’.



The Job entity has eight attributes: Job_ID, company, province, city, salary, postdate, summary, and joburl. The primary key of the Job entity is the Job_ID. The company attribute is a foreign key that allows the employer to edit that job. All other purely details of the job.



Joblist table



Joblist table as seen in XAMPP's phpMyAdmin: This manages the database of Jobs along with the attributes: posting ID (int), job title (text), company (text), city (text), province (text), salary range (text), postdate (int), summary (text) and the job url (text).



Back end data processing


For the backend of search queries, all the values submitted for search were collected, and tested if they were empty or not. If the values were not empty, the value would be passed into the query string. When this was done, the result was displayed.



Description of the front end dynamic support/ajax



We used AJAX for two things in our website. For populating the list of search results, and for going through the pages of the search results. For populating the search results, the PHP code we wrote collected the search criteria for the processing page. Then we used string concatenation to assemble the query, with the limit of results set to 10.



For moving through the pages, the PHP sent the same search criteria, but the result starting value was set to +10, so each time the button is pressed, it will move the page forward.



You can see the query building here



For going backward, it is similar. We reduced the number of the start post, but only if the post was greater than 10, so that the post start would not go into negative, throwing an error.
